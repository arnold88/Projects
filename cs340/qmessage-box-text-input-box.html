QInputDialog is provided for convenience, as a standard dialog. It's not customizable (except for the parameters passed to its static functions). Also, QInputDialog::getText() accepts empty strings and you cannot change this behaviour.

You'll have to write your own dialog if you want a different behaviour or a different look. Here are the minimal steps to write a project that features your own input dialog :

1/ Create a new project with a main window. In QDevelop the steps are : Project > New project. Select "Application with Main Window".

2/ Add a dialog to this project. In QDevelop the steps are : Project > Add a new element. Select "Dialog". A new .ui file is added to your project, which contains your dialog. Right-click the .ui file. Select "Subclass Dialog" and click on "New". The .cpp and .h files are generated and added to your project.

3/ Open the .ui file of the dialog in Qt Designer. Drop 1 line edit and 2 push buttons (label those "cancel" and "ok"). With the visual signals & slots editor, associate :
  • okButton.clicked() with Dialog.accept()
  • cancelButton.clicked() with Dialog.reject()

4/ In the source file of your main window, do :

Source code

#include "myinputdialog.h"

void MainWindowImpl::function() {
	MyInputDialog dialog(this, Qt::Dialog);
	if(dialog.exec() == QDialog::Accepted) {
		QString text(dialog.lineEdit->text());
		//do something with text
	}
}

Quoted from "mike phillips"

I cannot see where it tells me that I cannot adjust 'pos' and 'rect' for the widget.

If you look at the documentation of QInputDialog, you'll see that there's no public constructor, meaning : no public QInputDialog(...) function. So you cannot instantiate this standard dialog, meaning : you cannot do :

Source code

 
1

QInputDialog dialog; // illegal

The only way to invoke this dialog is through the static functions. Therefore, the only adjustments possible are those provided by the static function parameters.
